==================================================
TYPOGRAPHY
==================================================

Use only three type styles:

1. Display serif for hero/project titles:
Instrument Serif, Source Serif 4, or IBM Plex Serif

2. Body/UI sans:
Inter, IBM Plex Sans, or Manrope

3. Technical mono:
IBM Plex Mono or JetBrains Mono

Rules:
- Use serif only for large headings and rare editorial emphasis
- Use sans for all body text and UI
- Use mono only for labels, metadata, tags, figure labels, routes, controls, and annotations
- Do not use mono for paragraphs
- No more than three font families
- Strong hierarchy, comfortable reading width, no giant marketing text

SKEUOMORPHIC COMPONENTS
==================================================

Create reusable UI components that use physical cues with restraint.

1. PaperSheet
Use for significant content areas:
- Slight paper-color variation
- 1px graphite rule
- 2–5px subtle low-opacity shadow
- Very slight darker lower/right edge
- Optional clipped corner OR folded-document corner, never both at once
- Not every paragraph needs a card

2. MetalDataPlate
Use for project labels and compact metadata:
- Brushed warm gray metal appearance
- Fine dark outline
- Very subtle top inset highlight
- Engraved mono text
- At most one or two tiny screw heads per larger plate
- No fake excessive chrome
- Use for project IDs, labels, tools, roles, and section index

3. PhysicalButton
Use for real calls to action:
- Matte graphite or muted-metal surface
- 2–4px corner radius maximum
- Fine border
