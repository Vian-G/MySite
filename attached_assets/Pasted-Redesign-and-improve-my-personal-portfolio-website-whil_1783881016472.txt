Redesign and improve my personal portfolio website while preserving the existing “technical systems / robotics terminal” brand aesthetic. The current site already has a strong voice, but it needs to feel more credible, clearer, and more useful to internship recruiters, professors, and engineering collaborators.

Goals:
1. Keep the visual identity futuristic, minimal, and robotics-oriented, but reduce gimmicky terminal language where it hurts clarity.
2. Make the site feel like a serious engineering portfolio, not just a stylish landing page.
3. Improve readability, project proof, navigation clarity, and conversion.

Core changes:
- Keep the homepage hero, but rewrite it to be clearer and more human.
- Add a concise value proposition under my name: who I am, what I build, and what roles I’m targeting.
- Add prominent CTAs: View Projects, Download Resume, Contact Me.
- Add an email/contact link in the hero or header.
- Remove or hide any visible hosting/promotional banners like “Build for free.”
- Keep the “system” motif, but use it as accent language only, not for every important label.

Homepage structure:
1. Hero section
   - Name, title, short positioning statement
   - One-line summary of skills and goals
   - Clear CTA buttons
2. Featured projects section
   - Show 3 flagship projects directly on the homepage
   - Each card must include: project name, one-sentence problem statement, my role, stack, and 1 measurable result or achievement
   - Add a “View case study” link for each
3. Skills / capabilities section
   - Keep embedded, controls, perception, and mechanical categories
   - Make them easier to scan with cleaner typography and spacing
   - Reduce all-caps overload
4. About snapshot
   - Short paragraph about my background, interests, and what kind of work I want to do
   - Add location and current status
5. Contact/footer
   - Email, LinkedIn, GitHub, resume link
   - Simple and professional

Projects page:
- Turn it into real case studies, not just thumbnails or links
- For each project include:
  - Title
  - 2–3 sentence overview
  - Problem / objective
  - What I built
  - Tools/stack
  - Key challenges
  - Results / lessons learned
  - Photos, diagrams, renders, or videos if available
- Make project cards visually consistent
- Use strong hierarchy and readable spacing

About page:
- Make it more personal and credible
- Include my engineering interests, what motivates me, and what kind of robotics/embedded work I enjoy
- Add a short “currently exploring” section
- Add a headshot placeholder area only if no image exists, but make the layout still look complete without it

Resume page:
- Ensure the resume link is obvious and works
- Embed a clean preview if possible
- Add a plain “Download PDF” button with conventional wording

Design direction:
- Informational portfolio site, not a SaaS app
- Keep the dark, technical, system-console flavor, but refine it
- Maintain restrained color usage, strong typography, and clean spacing
- Use the theme as identity, not as a usability obstacle
- Reduce visual noise from repeated labels like REV_01, END_OF_FILE, STATUS, etc.
- Make headings and body text more readable
- Keep a premium, engineered feel

UX/accessibility:
- Use semantic HTML
- Keep one clear H1 per page
- Ensure strong color contrast
- Make nav labels conventional enough to understand quickly
- Ensure buttons look clickable
- Improve mobile layout and spacing
- Keep touch targets large enough
- Add visible focus states
- Make external links open in new tabs

Performance:
- Optimize images
- Defer non-critical JS
- Keep the page fast and lightweight

Important:
- Do not replace the whole identity with a generic startup template.
- Do not use cheesy gradients, glowing blobs, or generic AI-generated SaaS sections.
- Preserve the robotics/system personality, but balance it with clarity, substance, and professionalism.
- The end result should impress both technical peers and non-technical recruiters.
